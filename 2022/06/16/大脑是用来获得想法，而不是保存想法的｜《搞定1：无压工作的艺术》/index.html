大脑是用来获得想法，而不是保存想法的｜《搞定1：无压工作的艺术》

要将GTD这个概念应用起来才算真正理解了本书吧！虽然目前我面对的事情不至于用上里面推荐的所有管理方法，但也算接触到足够多的种类了，迫切需要更合理的日程规划！

今天读完后运用里面的收集->理清->组织->回顾->执行工作流更新了电子设备上的工作流App，受益匪浅。

在苹果系统里挑选了原生的三个软件用来实现整个GTD工作流，分别是：日历、提醒事项、备忘录。

  • 「日历」按常规用法，只记录特定时间点需要处理的事情，比如生日或者ddl，然后添加小组件到手机页面（放到微信的那一页就容易看见）。然后按照平日里接触频率高的事务类别，分为”生活、工作、健康、创作和学习“五款日历，同时开放订阅我国节假日调休和生日。
  • 「提醒事项」非常适合用来创建清单，格式上满足打勾和拖拽移动的需求，现在也更新了标签、备注、子项目和优先级等功能，其中标签可以跟”生活、工作、健康、创作和学习“五款日历对应。然后按需创建“收集箱、执行清单、等待清单、将来/也许、项目清单和复盘”6个列表。
    • 「收集箱」：收集任何想法，想到就写进去，也可以叫siri写进去。
    • 「执行清单」：将收集箱里的想法写成具体的行动。
      If 2min内可以做完的，比如回复一个面试邀请的邮件，就直接做掉；
      else if 需要亲自动手但比较复杂，比如跑模型，就丢进「项目清单」里，分步规划出具体行动；
      else if 不用亲自动手，可以指派他人但要跟踪进度的，比如指导本科毕设，就放进「等待清单」里；
      else 剩下的就是今天计划要做的具体行动了。
    • 「等待清单」：见上。
    • 「将来/也许」：就是一些灵感创作或愿望，想做但现在没空，比如学习Final Cut Pro。
    • 「项目清单」：这里先建一个单独的列表记录目前手头上有的所有项目，比如“7月前减肥到60kg”、“2022毕业答辩”、“完成win到mac的数据迁移”和“用GTD方法处理outlook邮箱（招聘邮件归档）”等，然后再为每个项目设定子项目或下一步行动，然后把今天要做的更新到「执行清单」里，其他的等待或者复盘事项也是各回各家。
    • 「复盘」：主要是检查和回顾两类。一些检查清单，比如谈薪的时候要问哪些问题，列出每次都要做的常规套路，方便查看；又或者是每周回顾，写周报的话都要写那一类的工作，并为这种重复的回顾单独设计循环提醒时间。
  • 「备忘录」这里记录一些平时需要用到的资料或者随手取用的参考文件，比如邮箱号、银行卡号、菜谱或者灵光乍现的小黄文之类的。进一步可以是年度规划、书单、影单之类的更新与整理。

或许你可能发现前两个「日历」和「提醒事项」已经能够把书里介绍的管理工具概括了，也不需要再多一些「备忘录」的边角料啥的。

但实际操作的时候发现，上面提出的工作流仅仅只是处理事务的优秀方案，并没有给上层思维或者整体的规划带来更高意义的提升。「备忘录」里面就可以记录一些类似的灵感和月度、年度、3～5年职业计划之类的。

书里有提到【自然计划模式】这个概念，分为：

  • 5楼：目标和原则
  • 4楼：愿景
  • 3楼：目标
  • 2楼：关注点及责任范围
  • 1楼：当前项目
  • 地面：当前行动

仔细看会发现，GTD工作流更多是在“地面和1、2楼“给我们提供帮助，往上与我们发展息息相关的更远更大的计划就不再适合使用这套方法了。靠的更多是洞见和经验，这一点需要大量的知识来深化和更新我的整个人生管理系统，这里先立个flag。

最后，还有一个很大的工程就是::归档::。书里给到一些建议，但同时提醒我们这一步并不像GTD工作流那样掌握概念后很快上手，需要自我的认识和长时间的探索及更新才会慢慢形成专属的系统。

从今年年初接触这本书开始，我就陆陆续续地思考如何构建个人知识体系，尝试过超深度嵌套本地文件夹、云端网盘（挂了20部漫画）还有搭建个人博客、GitHub等，虽然都还在试验阶段，但对归档的重视却是愈渐加深的。待我探索一番再更新发布相关文章或视频吧。
